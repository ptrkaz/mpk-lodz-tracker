# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Expo (React Native) mobile app showing live MPK Łódź vehicle positions on a MapLibre map. Single screen, no backend — the device polls the public GTFS-Realtime feed published at `otwarte.miasto.lodz.pl` directly. Spec lives in `docs/superpowers/specs/2026-05-01-mpk-lodz-tracker-design.md`, plan in `docs/superpowers/plans/2026-05-01-mpk-lodz-tracker.md`.

## Commands

- `npm run typecheck` — `tsc --noEmit`. Strict mode (`exactOptionalPropertyTypes`, `noUncheckedIndexedAccess`).
- `npm test` / `npm run test:watch` — jest-expo, runs all unit/integration suites.
- `npx jest path/to/file.test.ts` — single test file. Per-test: `npx jest -t 'partial test name'`.
- `npx expo prebuild --clean` — regenerates `android/` and `ios/`. Required after touching native config.
- `npx expo run:android` / `npx expo run:ios` — full native build + install + Metro.
- `npx expo start --android` — Metro only (assumes APK already installed). For pure JS reload.

Native folders (`android/`, `ios/`) are gitignored and regenerated by prebuild. Don't hand-edit them — see "Config plugin" below.

## Architecture

Layered, every layer testable in isolation:

```
api/        pure async fns (fetch + decode), no React, no stores
store/      Zustand stores: vehicles, routes, filter — no I/O
hooks/      orchestration: bootstrap (one-shot, cached) + polling (10s + AppState pause)
components/ presentational pieces (markers, chips, hints) — no data fetching
screens/    composition: MapScreen wires hooks + stores + components; FilterSheet ditto
i18n/pl.ts  every user-visible string lives here
constants/  endpoints, Łódź camera defaults, vehicle colors, TTLs
```

**Data flow.** On boot, `useGtfsBootstrap` reads cached `RoutesIndex` from `expo-file-system` (legacy API), refreshes if older than 7 days. `useVehiclePolling` fetches `vehicle_positions.bin`, decodes via `gtfs-realtime-bindings` (protobufjs under the hood), commits `Vehicle[]` to the store. `MapScreen` joins vehicles with the routes index *at render time* — vehicles can stream in before routes resolve, falling back to displaying `route_id`.

**Map markers.** Single `GeoJSONSource` + two `Layer`s (circle for the colored badge, symbol for the line-number text). `circle-color` and earlier `text-halo-color` use `match` expressions on `properties.type` (`'tram' | 'bus' | 'unknown'`). The `as unknown as string` casts on expression arrays are a known typing gap in `@maplibre/maplibre-react-native` v11 — don't try to "fix" them with stricter typing.

**Filter store keying.** `selectedRouteIds` is keyed by `route_id`, not line number. The same number can exist for tram and bus, so route_id is the only safe key. Display in the sheet groups by `activeTab` (tram/bus).

## Config plugin

`plugins/withCertumNetworkSecurity.js` is a custom Expo config plugin. It exists because the GTFS-RT endpoint's certificate chains to **Certum Trusted Root CA**, which is not in Android's default trust store — without intervention, every fetch fails with `TypeError: Network request failed`. The plugin:

1. Copies `plugins/assets/certum_root_ca.pem` to `android/app/src/main/res/raw/`.
2. Copies `plugins/assets/network_security_config.xml` to `android/app/src/main/res/xml/`. Config has `base-config cleartextTrafficPermitted="true"` (so Metro on `http://10.0.2.2:8081` works in dev) plus a `domain-config` for `miasto.lodz.pl` that adds the Certum cert as a trust anchor.
3. Patches `<application android:networkSecurityConfig="@xml/network_security_config">` into `AndroidManifest.xml`.

`expo prebuild --clean` will run the plugin and regenerate everything. **Never edit `android/app/src/main/res/{raw,xml}/...` or `AndroidManifest.xml` directly** — your edits will vanish on the next prebuild. Edit the source files under `plugins/assets/` instead.

If the cert ever needs refreshing: `echo | openssl s_client -servername otwarte.miasto.lodz.pl -connect otwarte.miasto.lodz.pl:443 -showcerts 2>/dev/null | awk '/BEGIN CERTIFICATE/{c++} c==3{print} /END CERTIFICATE/{if(c==3) exit}' > plugins/assets/certum_root_ca.pem`.

## Env vars

- `EXPO_PUBLIC_MAPTILER_KEY` — required for map tiles. Empty/missing → blank map (HTTP 403 from MapTiler). `app.config.js` reads it and surfaces it via `Constants.expoConfig.extra.maptilerKey`. Live in `.env` (gitignored). The `extra` field MUST be populated from `process.env` at config-resolve time — a literal `"${EXPO_PUBLIC_MAPTILER_KEY}"` in static `app.json` will NOT be expanded.

## Tests

`jest-expo` preset. Tests live alongside source as `__tests__/`. Real fixtures in `__fixtures__/`:

- `vehicle_positions.bin` — captured GTFS-RT snapshot. Used by `gtfsRealtime.test.ts` to exercise the real protobuf decoder. Regenerate with `curl -fsSL https://otwarte.miasto.lodz.pl/wp-content/uploads/2025/06/vehicle_positions.bin -o __fixtures__/vehicle_positions.bin` if the schema ever changes.
- `GTFS-mini.zip` — hand-crafted minimal GTFS zip with three routes (2 trams, 1 bus). Generated inline in the plan; rebuild from `docs/superpowers/plans/...` if needed.

Hooks are tested with `@testing-library/react-native`'s `renderHook` + `jest.useFakeTimers()`. UI components have no unit tests — `docs/manual-test.md` is the smoke-test checklist.

## Known sharp edges

- **`jest.mock` factory variable naming.** Babel-jest only allows out-of-scope vars in `jest.mock()` factories if they're prefixed `mock` (case-insensitive). Use `mockFetchVehiclePositions`, not `fetchVehiclePositionsMock`.
- **Reanimated babel plugin in tests.** Disabled via `process.env.NODE_ENV === 'test'` in `babel.config.js` because `react-native-worklets` peer dep init crashes Hermes in jest.
- **Expo's `__ExpoImportMetaRegistry` enumerable getter.** `jest.setup.ts` demotes enumerable getters on `globalThis` to non-enumerable to dodge a teardown crash where `Object.keys(global)` triggers lazy getters after the module registry is cleared. Don't remove that loop without re-running the full suite.
- **`expo-file-system/legacy`.** v19+ split the API. We import from `/legacy` and `tsconfig.json` has a `paths` alias pointing the type resolution at `build/legacy/index.d.ts` (the source `.ts` trips `exactOptionalPropertyTypes`).
- **Non-spec deviation: no `expo-router`.** Spec mentions it; we use plain `App.tsx` rendering `<MapScreen />`. Documented at the top of the plan.

## Spec gaps the implementation does not yet cover

These are real spec requirements that were deferred — fix them properly rather than building around them: marker tap callout (`Tramwaj 8 · 2s temu`), bearing-based marker rotation (already decoded into `Vehicle.bearing`, just not rendered), offline indicator (50% opacity + `pl.map.offline` toast), hide locate-me FAB on permission denial, full Polish plural rule (1 / 2-4 / 5+), `expo-system-ui` install for `userInterfaceStyle: automatic` on Android.
